关于多态

java中 父类的类型决定了子类的类型 当调用函数需要传入一个参数 这个参数的类型是声明好的 传入的时候应当是这个类型的参数或者它的子类，这就是多态

python中则 无需关心类型 传入什么类型的参数就是什么类型 指向的是当前传入的对象

继承的一些注意点：

1.类名调用实例方法需要传入self和其他参数

2.父类类名调用init方法与1 一样

3.多继承 super()调用父类方法应当把参数全传入

4.super()

4.super()调用父类方法 重名的话默认只调用第一个

关于new方法和init方法

通俗的理解就是 没有对象需要new对象

new(cls)中cls是必要参数，代表要实例化的”类对象” 也就是当前创建的类 new方法与 init方法一样 是自动执行的 必须要有返回值 默认是object.new(cls)创造出来的实例 init（self）中的self就是该方法返回的实例对象 所以new方法会比init先执行

设计模式:单例 是重写new方法得来的

生成器、迭代器、装饰器、闭包

迭代器：它是一个可以记住遍历位置的对象 ，每迭代一次都会返回对象中下一条数据 一直向后 直到迭代完所有数据为止。可以迭代的对象包含迭代器，iter()方法可以获取迭代器 next()方法可以不断调用来不断获取下一条数据。 可迭代对象不一定是迭代器，迭代器一定是可迭代对象。

1.一个具有iter方法的对象，就是一个可迭代对象 iter方法需要返回一个迭代器

2.必须实现了next方法的才能是迭代器（因为迭代器的作用就是通过next向后访问元素） 所以可迭代对象中必须返回一个迭代器才能遍历 简单来说就是:能被遍历的一定是一个实现了iter和next的迭代器但不一定是可迭代对象，迭代器一定是可迭代对象，但可迭代对象不一定是迭代器。

3.综上所述 迭代器一定能被遍历 可迭代对象不一定 。

生成器：一边循环，一边计算的机制，称为生成器(generator) ,列表推导式的中括号，变成普通括号那么此时就是生成器，生成器是一种特殊的迭代器。注：列表推导式的列表是一个可迭代对象，而改括号的方式得到的是迭代器(生成器)。

1.使用了yield关键字的函数不再是函数，而是生成器。

2.使用next调用生成器，执行顺序：

  • 如果是第一次执行，则从def代码块的开始部分执行，直到遇到yield为止，并且把yield关键字后的数值返回，当做next()的返回值, yield在这里有了return返回值的作用。
  • 如果不是第一次执行，则从上一次暂停的位置执行(即从上一次yield关键字的下一个语句开始执行)，直到遇到下一次yield为止，，并且把yield关键字后的数值返回，当做next()的返回值

\3. 对象.send()同样可以唤醒 yield 并且可以传入一个值 这个值就等于 yield xx(result=yield xx)

\4. nextd(对象)等价于 对象.send(None) 需要注意的是：next()可以随时使用，send()不能用于第一次唤醒 只能用于yield 停止后唤醒。

闭包：函数内部嵌套一个函数，内部函数可以使用外部函数的参数或变量。最终返回这个内部函数。

1.闭包内部函数无法修改外部函数的变量 nonlocal可以访问外部非全局变量。

2.闭包可以解决多个变量 不同组合的变化情况 提高代码复用 如 一次函数 y=kx+b 不同斜率和不同偏移以及不同x 最终得到的不同的y 。

装饰器：是利用闭包在目标函数执行时进行一下功能拓展。它是将装饰的目标函数作为引用，然后在内部函数中执行，并且在执行之前进行功能拓展。

\1. @xxx装饰的目标函数Test，那么@xxx的作用就是将 Test变成 xxx(Test) 调用装饰后的Test ( Test( ) )等价于 xxx(Test)( )

2.带参数的装饰器函数调用等价于 xxx(参数)(Test)( ).

3.两个个装饰器同时装饰一个函数实质上是外面一层对里面装饰后的装饰。一层装饰器装饰后的函数实质上等价于装饰器内层函数，调用也等价于调用内层函数，两个装饰器等于装饰了内层装饰器的内层函数，其他多层装饰依此类推。

4.类装饰器，重写call方法

==、is 、深浅拷贝 可变类型、不可变类型详解

a=[1,2,3] b=[1,2,3] 打印a和b的地址不同，a==b为True，a is b 为False。

浅拷贝分为两种：(1) c=a ，这种是直接将a的引用复制一份给c，也就是指向a相同的地址 (2) c=copy.copy(a) ,这种是复制a的数据新建一个列表，地址会变，但内部元素的地址依然跟a一样。

深拷贝：简单来说就是递归拷贝，拷贝到一个新的地址，并且引用。深拷贝在递归拷贝时遇到不可变类型时是像浅拷贝第一种情况指向地址，而不是拷贝到新地址。

可变类型：一般来说可变类型最底层的数据是引用的不可变类型，它可以不断增减引用，多数时候内部的元素是会指向多个不可变类型的。

不可变类型：我们可以参考牛顿第一定理来理解，惯性是物质本身具有的，与运动状态无关。不可变类型，只要有引用，那么这个地址的数据就会一直存在，与引用方式无关，与引用多少次无关，并且它只会在内存中存在一个，直到引用为零。

tips：切片和字典的.copy( )也是浅拷贝的第二种情况

上下文管理器

任何实现了enter和exit方法的对象都可以叫做上下文管理器，上下文管理器可以使用with关键字

open()就是一个上下文管理器，也可以自定义实现。 其中，enter 方法是打开文件 exit方法是做一些清除工作，故无需调用close()去关闭文件。

python中使用@contextmanager 去装饰一个生成器也可自定义上下文管理器 yield之前的是执行enter的内容 yield之后的是执行exit中的操作。
